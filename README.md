# GDG DGU Web/App 파트 Codelab Repository

이 레포지토리는 **GDGoC DGU Web/App 파트**의 실습 중심 학습 활동인  
**파트별 코드랩(Codelab)** 과제를 관리하기 위한 저장소입니다.

Web/App 구성원들이 Google 기술 기반의 실습을 직접 수행하며  
Web/App 개발 역량을 성장시키는 것을 목표로 합니다.

---

## 💡 Web/App 파트 Codelab이란?

Web/App 파트 구성원들이 Google Codelabs 중심의 실습 과제를 수행하며  
**직접 코드를 작성하고, 오류를 해결하고, 경험을 공유하는 학습 방식**입니다.

각 주차마다 Web/App 파트 Core가 관련 기술 스택에 적합한 Codelab을 선정하고,  
구성원들은 이를 수행하며 기술 이해도를 높여갑니다.

---

## 🚀 진행 방식

### 1. **Core의 주차별 Codelab 선정**

- Web/App 파트 Core가 파트 로드맵에 맞는 **Google Codelabs 과제**를 선정합니다.

### 2. **개별 Codelab 수행**

- 구성원들은 주간 활동 시간 전까지 Codelab을 직접 수행합니다.
- 발생하는 에러나 궁금증은 언제나 질문 가능합니다.

### 3. **Codelabs & Mini Seminar**

정규 활동 전에 진행되는 Web/App 파트의 실습 공유 시간입니다. Codelabs 공유 후에는 Web/App 내 Mini Seminar를 진행합니다. Mini Seminar는 매주 2명의 파트원이 발표하며, 발표 순서는 사전에 정해진 일정에 따라 진행됩니다.

- 각자 구현한 코드/해결 과정 공유
- 오류 해결 과정 및 인사이트 교환
- Mini Seminar 진행 및 질의응답

> 실습 기반 학습을 통해 Web/App 파트 내부의 **협력적 개발 문화**를 만들어요!

## 📁 Repository 구조 및 사용 방법

### 1. **브랜치 생성 또는 Fork**

- 작업 전 **본인 이름으로 브랜치 생성** 또는 **저장소 Fork** 후 작업합니다.

### 2. **폴더 구조 규칙**

저장소에는 다음과 같은 주차별 폴더가 생성됩니다:

```
webapp-activity-1/
webapp-activity-2/
webapp-activity-3/
…
```

각 주차 폴더 안에 **본인의 이름 폴더**를 생성하여 그 주차 결과물을 업로드합니다.

예시:

```
webapp-activity-1/
└── 이정선/
├──── README.md
└──── 코드 파일들…
```

### 3. **Pull Request 제출**

- 작업 완료 후 PR을 생성합니다.
- PR 제목은 `[ACTIVITY-NN] 이름`의 형식을 따릅니다.
- 제공된 PR 템플릿을 따라 작성해주세요.
- Reviewer 지정:
  - `@dongmin0204`
  - `@leejs0823`

### 4. **제출 마감 및 관리**

- 제출 마감 기한은 매주 Web/App 파트 활동 시 안내됩니다.
- PR 생성 후 **노션에 PR 링크 제출**해주세요.
